## 4. Kardeş Satır Senkronizasyonu

**Kapsam:** `metrajolusturcetvel/index.js`

Tüm kardeş revize talep satırlarının (DB + form) birlikte hareket etmesi:

- **Kum saati (⏳)** herhangi bir kardeşe basıldığında → aynı ebeveyne ait tüm kardeş DB satırları pending, tüm form satırları `submitted_for_approval` yapılır
- **Tik (✓)** herhangi bir kardeşe basıldığında → tüm kardeş DB satırları draft, tüm form satırları `draft` yapılır
- **"+" butonu** tıklandığında → mevcut pending DB kardeşleri otomatik draft'a döndürülür; mevcut `submitted_for_approval` form satırları da draft'a döndürülür

## 6. "+" Sonrası Yeni Form Satırlarının Editable Olmaması Düzeltmesi

**Kapsam:** `metrajolusturcetvel/index.js`

**Sorun:** Daha önce `submitted_for_approval` olan form satırları varken "+" butonuna basıldığında:
- Pending DB kardeş yoksa `handleRevertPendingToDraft` çağrılmıyordu
- Eski submitted form satırları olduğu gibi kalıyordu
- Yeni satır draft ekleniyordu ama eski satırlar non-editable görünüyordu

**Fix:** "+" onclick'teki `setRevizeForms` inline olarak mevcut `submitted_for_approval` satırları `draft`'a döndürür.

```js
const existing = (prev[node.id] ?? []).map(r =>
  r.status === 'submitted_for_approval' ? { ...r, status: 'draft' } : r
)
return { ...prev, [node.id]: [...existing, newRow] }
```

## 7. DB Satırı Pending→Draft Revert Sonrası Editable Olmaması Düzeltmesi

**Kapsam:** `metrajolusturcetvel/index.js`

**Sorun:** "+" ile auto-revert yapıldığında 2.1 (DB satırı) `pending → draft` oluyordu, ancak `childEditValues[2.1.id]` initialize edilmemişti. `isChildEditable = true` ama `childVals = null` olduğu için input alanları yerine static text gösteriliyordu.

**Fix:** `handleRevertPendingToDraft` içine revert edilen satırlar için `childEditValues` initialization eklendi:

```js
setChildEditValues(prev => {
  const updated = { ...prev }
  toRevert.forEach(l => {
    if (!updated[l.id]) {
      updated[l.id] = {
        description: l.description ?? '',
        multiplier: ..., count: ..., length: ..., width: ..., height: ...
      }
    }
  })
  return updated
})
```

## 9. `isRevised` Dalında Toggle Mantığı Düzeltmesi

**Kapsam:** Her iki dosya

**Sorunlar:**
1. Gri "geçersiz kılınan" satır: `expandedOnayKarti && showAllOriginals` ikisi birden gerekliydi → sadece `showAllOriginals` yeterliydi
2. Onaylı çocuk satırlar (örn. 2.1): `expandedOnayKarti` ile gizleniyordu → her zaman görünür olmalı

**Fix:**

```jsx
// Gri satır: sadece showAllOriginals'a bağlı
{showAllOriginals && ( ...gri satır... )}

// Çocuklar: approved her zaman, ignored showAllOriginals ile, diğerleri expandedOnayKarti ile
{node.children.filter(c => {
  if (c.status === 'approved') return true
  if (c.status === 'ignored') return showAllOriginals
  return expandedOnayKarti
}).map(child => renderOnayRow(child))}
```
